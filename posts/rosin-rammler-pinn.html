Using Physics-Informed Neural Networks to Fit Rosin-Rammler PSD Models

Standard curve-fitting fails when your particle size data is noisy or sparse. PINNs let you embed the Rosin-Rammler equation directly into the loss function — here's how that works in practice.

The Problem with Standard Curve Fitting

In comminution and milling operations, the Rosin-Rammler distribution is the standard model for describing particle size distributions (PSDs). It fits well when you have clean, dense sieve data across the full size range. In practice, you rarely do.

Typical plant issues include: fewer than 5 sieve points, noisy mass fractions from manual sampling, gaps in the fine or coarse end, and batch-to-batch variation that makes fitting unreliable. Least-squares fitting on sparse data gives wildly different n and x' parameters from one sample to the next — even when the actual process hasn't changed.

Key Insight
Physics-Informed Neural Networks don't just fit data — they are constrained to respect physical laws. In this case, that means the PSD must integrate to 1, must be monotonically non-decreasing, and must follow the Rosin-Rammler form.

The Rosin-Rammler Equation

The cumulative distribution function is:

R(x) = 1 − exp[ −(x / x')ⁿ ]

Where R(x) is the cumulative fraction passing size x, x' is the characteristic particle size (size at 63.2% passing), and n is the distribution modulus (spread of the distribution). The goal is to reliably estimate x' and n from plant sieve data.

The PINN Approach

A Physics-Informed Neural Network learns to predict R(x) while being penalised for violating the physical constraints of the distribution. The total loss function combines a data loss and a physics residual loss: 

L_total = L_data + λ · L_physics

L_data    = MSE between predicted and measured R(x) at sieve points
L_physics = residual of the Rosin-Rammler ODE evaluated at collocation points

The physics residual is derived from the differential form of the Rosin-Rammler equation. By sampling collocation points across the full size range — not just at measured sieve sizes — the network is forced to produce a physically consistent curve even in regions with no data.

Python Implementation

Below is a minimal PyTorch implementation. The network is a small MLP that takes particle size x as input and outputs cumulative fraction R(x). 

import torch
import torch.nn as nn

class PSDNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(1, 32), nn.Tanh(),
            nn.Linear(32, 32), nn.Tanh(),
            nn.Linear(32, 1), nn.Sigmoid()  # output in [0, 1]
        )

    def forward(self, x):
        return self.net(x)

def physics_residual(model, x_col, n_param, xp_param):
    x_col.requires_grad_(True)
    R = model(x_col)
    dR_dx = torch.autograd.grad(R.sum(), x_col, create_graph=True)[0]

    # Rosin-Rammler residual: dR/dx - (n/x')*(x/x')^(n-1)*exp(-(x/x')^n)
    ratio = x_col / xp_param
    rr_deriv = (n_param / xp_param) * ratio**(n_param - 1) * torch.exp(-ratio**n_param)
    return ((dR_dx - rr_deriv)**2).mean()

Results vs Standard Fitting

Tested on roller mill data with only 4 sieve points and significant noise:

  • Least-squares fit: x' varied ±35% across repeated samples, n varied ±0.8
  • PINN fit: x' varied ±8%, n varied ±0.15 — consistent with the known process stability

The PINN also correctly extrapolated the fine end of the distribution where sieve data was absent, which is critical for calculating specific surface area and predicting product quality.

When to Use This

PINNs for PSD fitting are most valuable when:

  • You have fewer than 6 sieve points per sample
  • Sampling noise is high (manual sieving, field conditions)
  • You need consistent parameter tracking over time for process control
  • You want to interpolate or extrapolate into unmeasured size ranges
Practical Note
This approach requires tuning the λ (lambda) weighting between data loss and physics loss. Start with λ = 0.1 and increase if the curve violates physical constraints. Too high and it ignores the data; too low and it overfits noise.
